Programming languages are a big hobby of mine! Here's a few that I made:

Wisp (2013)

My first real Lisp! Written in Haskell during the Winter 2013 batch at Recurse Center (at the time called Hacker School). I had already messed around a little bit with Common Lisp due to StumpWM and had just been introduced to SICP. Wisp was intended to be the scripting language for a player-programmable MUD I was writing, but it quickly became the focus of the project. Semantically it's a fairly standard Lisp-1, but the implementation is a little odd since (a) it's in Haskell, (b) it runs inside a text-based multiplayer game, and (c) I had no idea what I was doing! One slightly unusual feature though was automatic currying.

Shimmre (2014)

A self-describing parser generator language inspired by OMeta and written as a blog post in literate CoffeeScript (because at my last job the CTO said CoffeeScript was the future; I guess the subsequent explosion of languages-that-compile-to-JavaScript proved him sort of right). You would write a grammar in Shimmre and provide transformation rules in a host language, and Shimmre would generate host language code. It had working JavaScript and Ruby backends. Shimmre was my first "compiled" language, and IMO it's still kind of cool. I'll probably rewrite it in plain JS one of these days.

Gus (2015)

I wanted to write another Lisp, in C this time. Gus is a single-C-file Lisp interpreter named after a coworker's cute dog. It uses a mark-and-sweep garbage collector that I more or less copied conceptually from Ian Piumarta's Maru.

Mattie (2016)

Kind of a weird experimental string-centric esolang, based on parsing expression grammars, like Shimmre. Mattie (named after my dear cat) was written in Scheme with a syntax based on Shimmre's metagrammar. It's entirely based around transforming strings. One of those languages that makes you go "is this really Turing complete?"

Lisa (2020)

Also known as Lips. Originally written as a FORTH-like stack language, it turned into another Lisp written in C, but this time with much cooler features: copying garbage collection, a self-hosting threaded code compiler, actual compile-time macros, and a stripped-down syntax based on just 5 special forms. Still in development.

K2 (2022)

A microKanren derivative written in Lisa that came out of my research into relational languages at Oso. Being based on microKanren it's very minimal. The main innovation is to represent result streams uniformly as Church-encoded lists, which removes a layer of runtime reflection; on the other hand it creates more closures, so the performance picture is unclear. Even so i think it's a very aesthetic approach to take! It would be less tolerable in Scheme, but Lisa's lambda syntax is so clean it's basically transparent.
